发表于2022-04-02|更新于2022-06-19|hexo
|阅读量:

前言

此篇博客主要为 在使用hexo与markdown中向博客中插入本地图片之后，出现本地运行后在相关网页打不开图片的问题 提供 一个较为详细的解决办法，以及原因分析

问题原因分析

根本原因在于 本地图片的存储路径与使用 hexo -generate 命令后在 public 目录下的生成的相应本地静态博客文件夹中的 index.html 文件中对于 实行引用图片操作的相关代码

 <div class="card-content article-card-content">
            <div id="articleContent">
                <p><img src="public/2022/04/02/test/image_1.jpg" alt="image_1">

所生成的图片路径不相匹配（这里之所以使用不相配。而没有使用不一致，是因为在我后续的实验中发现即使路径不同，也能正常显示，本人能力有限，没能真正理解）

解决问题方法

  1. 首先 在 _config.yml 文件中 把代码post_asset_folder: false 中的false改成true

  2. 在 hexo bash 中执行

npm install https://github.com/CodeFalling/hexo-asset-image --save 我们虽然在source_posts文件夹下写了md文件，也有了图片资源文件夹存了图片，但从我们前面typora中的设置不难知道，实际上md文件中的图片路径都是相对路径（./${filename}）。而实际网上看到的博文显然不是md文件，而是html文件，从md到html的转变就是hexo帮我们做的，还记得hexo g命令吗？就是干这个的。转换后的html文件在public目录下，路径是通过日期指示的,而此插件就帮助我们实现从相对路径到html中绝对路径的转换。
