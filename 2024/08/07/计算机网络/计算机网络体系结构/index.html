计算机网络体系结构

发表于2024-08-08|计算机网络
|阅读量:|评论数:

计算机网络的定义

计算机网络是一个将众多分散、自洽的计算机系统，通过通信设备和线路连在一起，通过软件实现资源共享和信息传递的系统。计算机网络由若干节点(node)和链路组成。节点可以是 通用计算机，路由器，集线器，交换机。异构的网络之间可以通过路由器连接成为互联网(internet)。

  • 软件：在计算机网络中负责管理和协调资源，实现资源共享和数据传递。例如
    网络协议栈(TCP/IP)，网络操作系统(Windows/Linux Server)，客户端软件(Chrome，电子邮件客户端)。
  • 路由器：用于连接异构的网络，传递数据。（工作在网际层）
  • 集线器：连接多个以太网设备，使他们作为一个网络段工作。（工作在物理层，相当于有多个端口的中继器，通过双绞线或者光纤连接多个以太网设备到同一个物理介质来组成同一个网络段，组成一个共同的冲突域）
  • 桥接器：将两个相似的以太网段连接成一个逻辑上的以太网段。（工作在数据链路层，可以隔离形形成两个冲突域。）
  • 交换机：连接多个设备，使用MAC来传递数据帧。（一般工作在数据链路层，实质是多个端口的桥接器，每个端口都是一个独立的冲突域。）
  • 以太网：以太网实现了网络上多节点发送信息的想法，每个节点需要获得信道和电缆才能传递信息，有时也叫以太。每个节点拥有一个全球唯一的48位的MAC地址，用来节点之间相互鉴别。由于以太网十分普遍，以太网卡基本都会继承进计算机主板。

计算机网络的组成

从组成部分来看分为：硬件，软件,协议。

  • 硬件指具体的主机(端系统)，通信链路(如双绞线，光纤)，交换设备(路由器，交换机)，通信处理机(如网卡)。
  • 软件就是指提供具体服务的程序运行的进程。
  • 协议是计算机网络的核心，规范了网络数据传输的规范。

从工作方式看分为：边缘部分,核心部分。

  • 边缘部分是所有连接到互联网上，直接供用户使用的主机，提供通信与资源共享。
  • 核心部分由大量异构的网络和连接这些网络的路由器组成，为边缘部分提供连通性额数据交换服务。

从功能分为：通信子网，资源子网。

  • 通信子网由各种传输介质，通信设备，网络协议组成，提供数据传输，存储服务。
  • 资源子网是提供共享服务的软件及其设备的集合，像网络用户提供软件，硬件，数据资源的服务。

计算机网络的功能

  • 数据传输：最基本和重要的功能。
  • 资源共享
  • 分布式和处理
  • 提高可靠性
  • 负载均衡

电路交换、报文交换、分组交换

电路交换

优点：

  • 传输时延小
  • 有序传输
  • 无数据传输冲突，专用信道
  • 既可以数字信号也可以模拟信号
  • 实时性强
  • 控制简单

缺点：

  • 建立时间长
  • 信道利用率低
  • 单点风险高，灵活性差
  • 兼容性差，难以规格化
  • 难以实现差错控制，健壮性差

报文交换

优点：

  • 无需建立连接
  • 信道利用率较高
  • 动态分配节点路线，灵活性好
  • 可以多播

缺点

  • 转发，传输时延较高
  • 缓存开销大（报文大小没有限制，采用存储转发，将后续到来的报文缓存在发送队列里）

分组交换

相较于报文交换：分组长度固定，流水线操作，额外信息大，一个报文的不同分组可能会失序，丢失，重复。


计算机网络分类

从分布范围看

  • 广域网(WAN)，互联网的核心部分，长距离传输，高速链路，。
  • 城域网(MAN)，局域网(LAN)。以太网技术。传统上局域网使用广播技术，广域网使用交换技术。

    现代以太网使用交换机来管理广播域，减少不必要的广播。
    交换机会根据MAC地址表进行帧转发，只向特定端口发送数据包。

  • 个人局域网(PAN)，通常为无线技术连接的网络。

从拓扑结构来看

指网路中节点与通信线路之间形成的几何结构。

  • 总线型：所有节点共享单根信道。建网容易，线路简单。对线路故障敏感，需要介质访问控制，在重负载时信道效率不高。
  • 星型：每个终端都以单独的线路与中央设备相连。中央设备一般是路由器与交换机。单点风险高。
  • 环形：如令牌环局域网。可以是单环，也可以是双环，环中信号是单向传输的。

计算机网络的性能指标

  • 速率：连接到网络上的节点单位时间传送数据的多少。
  • 带宽：最高数据率。
  • 吞吐量：单位时间内通过某个网络的数据量。
  • 时延：数据(报文，分组)从网络的一端到达另一端的时间。
    包括：
    发送(传输)时延：将数据推向链路上需要的时间。
    传播时延：比特在两端信道上传播需要的时间，与分组长度无关。
    处理时延：在交换节点为存储转发所需操作(分析首部，差分数据，处理差错，查找路由等)的时间。
    排队时延：交换节点会缓存后续到来的分组或报文，在发送队列中的排队时间。

计算机网络的分层结构

原则：

  • 每层实现独立功能，功能定义独立于实现。
  • 各层之间通过接口交流。
  • 下层对上层独立，上层单向调用下层服务。

PDU(Protocol Data Unit) = SDU(Service Data Unit) + PCI(Protocol Control Information)

对等实体之间传送的数据单元称为该层的PDU。

SDU即为服务数据。PCI即为协议控制信息。

PCI：
物理层：比特流
数据链路层：帧
网络层：分组
传输层：报文段

计算机网络协议，接口，服务

协议：

在对等实体之间传输数据时所要遵循规则的集合。在分层结构中强调的是水平关系之间的实体的抽象交互规则。不对等实体时间没有协议一说。

由 语法，语义，同步组成。
语法：数据与控制信息的格式，决定数据如何存放，到达后如何解析。
语义：发出何种控制信息(发送方)，做出何种动作(发送方发送后与接受方接收后)，做出何种应答(接收方)。与与语法相比，语义更强调根据静态的符合语法的数据做出何种动态的交互，侧重对等实体间的具体交互规则。
同步：执行各种操作，事件发生顺序的说明。

接口：

同一节点内相邻两层之间交换信息的逻辑接口称为 服务访问点(Service Access Point)。第n层的SAP就是该层可以提供给上层的服务。

服务：

上层可以调用紧邻下层的功能，是垂直的。通过交换服务原语(请求，指示，响应，证实)来达到服务的使用。

三者关系

  • 面向连接的服务：连接建立需要通信链路上的节点分配相关资源(如缓冲区)。
  • 可靠服务：指使网络具有 检错，纠错，应答机制，保证数据准确，有序达到。
  • 不可靠服务：数据的准确与可靠需要协议之外的应用或者用户来保证。

ISO/OSI参考模型

OSI(Open System interconnection) 第三层为通信子网，为联网而附加的通信设备，上四层为资源子网。也就是可以进行逻辑处理的计算机系统。

组成：

OSI与TCP/IP

区别：

  • OSI现有模型，协议，通用性好，适合描述各种网络。TCP/IP不适合其他非TCP/IP网络。
  • OSI在网络层支持无连接和面向连接两种通信，在传输层只提供面向连接的服务。
  • 而TCP/IP认为可靠性是端到端的问题，因此在网际层只有一个无连接，而在传输层有两种。

定义 是一种硬件设备，它通过无线电信号为设备提供无线网络接入，并且通常连接到有线网络（如局域网），实现无线设备与有线网络的桥接。 是一个能够提供无线网络接入的地点或服务。它通常是由设备（如智能手机、便携式热点设备、路由器）通过Wi-Fi信号创建的无线网络。
用途 扩展和管理无线网络 提供便携式互联网接入
功能 连接有线和无线网络:AP通过网线连接到有线网络（如交换机），然后将有线网络信号转换为无线信号，供无线设备（如手机、笔记本）连接。
扩展网络覆盖范围:在大型网络中，多个AP可以组合使用，扩大无线网络的覆盖面积。
管理与控制:企业级AP通常具备高级功能，如用户管理、网络安全、访问控制、VLAN支持等。
数据中继：AP在无线设备与有线网络之间中继数据包，确保数据流畅传输。		 便携式网络接入: 热点通常用于为移动设备（如手机、平板）提供互联网连接，特别是在没有固定网络的情况下。
共享移动网络: 许多手机可以通过共享其移动数据连接创建临时Wi-Fi热点，为其他设备提供互联网接入。
公共Wi-Fi服务: 许多公共场所（如咖啡店、机场、酒店）提供的Wi-Fi热点，供客户或游客使用。
典型应用 企业网络、家庭网络 公共Wi-Fi、手机热点
连接方式 有线连接到交换机或路由器 无需有线连接，直接创建无线
供电方式 电源适配器或PoE 设备电池或电源
设置复杂性 需要专业设置和管理 简单设置，用户友好
管理与控制 支持高级网络管理 基本功能，无高级管理
覆盖范围 大范围覆盖，多设备连接 小范围，有限设备连接
