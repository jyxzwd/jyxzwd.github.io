解决hexo图片问题

hexo 原生图片问题，路径不能正确解析

  • 更改根目录 _config.yml : post_asset_folder: true
  • 再写post时， 在与 post_name.md同级目录下创建一个同名目录: post_name，把图片放在里面。
  • package.json 中添加插件依赖 "hexo-asset-image": "github:CodeFalling/hexo-asset-image" npm install
  • 但此时 md 文件中图片的路径一般都是 ./post_name/xxx.png, hexo 解析后生成的 html 文件中生成的图片路径会多一层 /post_name

如何做到在编写md文档时能正确显示图片，发布的时候又能正确显示图片呢 使用uTools uTools 中的 markdown 插件编写 md 文档，然后导出，他在导出文档的时候，会自动在与文档同目录下导出 各张图片，此时只需要把md文档提到上一层目录即可


hexo deploy

配置 根目录下的 _config.yml

然后 hexo deploy

hit: 要提前配置好ssh密钥 参照 上文 ssh 配置


github actions github action quick start

在git项目下新建 .github/workflows/demo.yml

自动部署到 githubPage

同样要配置ssh密钥，在 githubPage仓库里 deploykey 添加 公钥， 在 actions 里的 secret里添加 HEXO_DEPLOY_KEY ，值为 私钥

在 commit push 之后就会自动 执行 (感觉很像 docke compose)

部署到远程服务器 使用 ssh-deploya action

同样需要配置 BLOG_SSH_PRIVATE_KEY 私钥
